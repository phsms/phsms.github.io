Como criei este site (e por que o elementor deu zebra)

Como você pode ver, esse site está hospedado no githubpages e apresenta o seu conteúdo através do HTML estático, sem banco de dados para o que está aqui, agora. Mas, fato é que este site foi feito no WordPress que está rodando em um ambiente local e o que está sendo exibido aqui é apenas a versão HTML/CSS/JS que foi exportada desse site wordpress local.

O plano inicial

Eu sabia que não iria realizar atualizações constantes nesse site, por experiência própria, uma rotina de atualizações não é meu forte, então eu pensei que poderia manter o sistema funcionando de forma estática e hospedá-lo no GithubPages foi uma boa ideia, já que não há tanta necessidade de manutenção e sem risco de invasão de banco de dados (já que ele só existe localmente).

E como ia ser estático, lembrei imediatamente da internet dos anos 1990 e 2000, com a estética característica dos Geocities e Altavista, era esse o plano. Fiz um protótipo de tecnologia para testar o processo de exportar um site WP para html e após testar alguns plugins, gostei do Simply Static.

A tentativa com o Elementor

Minha primeira tentativa foi usando o Elementor para construir o tema, mas na hora de exportar o resultado criado no Elementor para HTML usando o Simply Static, o sistema demorava uma eternidade, e quando eu ia investigar o arquivo exportado, eu notava a a presença de centenas de arquivos desnecessários, que estavam lá apenas por fazerem parte do Elementor.

Testei com um tema padrão e o resultado foi muito mais rápido e eficiente. O grande volume de arquivos sem necessidade e a demora para exportar a versão estática me fizeram desistir do Elementor e partir para a abordagem mais tradicional do tema próprio.

Criando o tema WP

O processo de criação do tema WP foi bem simples, peguei o arquivo HTML/CSS estático do site do Motorhead de Gabriel Fleig e transformei em tema, separando o arquivo index.html em head.php, index.php, footer.php e inserindo as funções dinâmicas do wordpress nesse processo de separação e montagem. Depois criei o single.php e adicionei o cabeçalho obrigatório no style.css: 

/*
Theme Name: 1990fleig
Author: Fleig
Description: Tema WordPress baseado no HTML/CSS original de Fleig.
Version: 1.0
License: GNU General Public License v2 or later
*/

Adicionei algumas funções no functions.php para registrar o menu, personalizar alguns recursos e facilitar a inserção de alguns conteúdos e voilá!, o site estava pronto. Restando apenas inserir o conteúdo e exportar usando o Simply Static.

Hospedagem no GithubPages

Uma vez exportado, o resultado final do Simply Static é uma pasta com um conjunto de arquivos dentro, HTML, CSS e as imagens, tudo que eu preciso fazer com isso é enviar para o repositório do Github, e ele se encarrega de publicar.

O pulo do gato é que a configuração do repositório no github precisa seguir uma convenção, ao criar um novo repositório tem que usar a nomenclatura: meu-usuario.github.io. Quando se cria um repositório com o nome do usuário seguido de .github.io o github entende que deve criar ali a raiz de um site e não apenas o código avulso.

Com essa abordagem, você tem a facilidade do WordPress rodando seguro no seu PC local, e a performance do HTML estático rodando na web. O site não cai, carrega rapidinho e não custa nada.
